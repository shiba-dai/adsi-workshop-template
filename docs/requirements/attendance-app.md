# 勤怠管理 Web アプリ — 要求仕様（確定）

## 概要

50名規模（SMB向け）の組織で使用する勤怠管理 Web アプリケーション。
社員の出退勤打刻、勤怠履歴閲覧、月次承認、休暇管理、月次レポート出力を提供する。

---

## ロール

| ロール | 説明 | 備考 |
|--------|------|------|
| 一般社員 | 自身の打刻・履歴閲覧・休暇申請・月次提出 | — |
| 管理者（管理職/ライン長） | 部下の勤怠確認・月次承認・有給承認・レポート出力 | 一般社員の機能も利用する（権限積み上げ型） |
| システム管理者 | 社員マスタ・部署・カレンダー管理・システム全体の設定 | — |

- 管理者は有給・月次勤怠ともに**自己承認**が可能

---

## ユーザーストーリー

### 一般社員

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-1 | 社員として、出勤打刻したい | Web画面のボタン1クリックで出勤時刻が記録される。1日1回のみ。 |
| US-2 | 社員として、退勤打刻したい | Web画面のボタン1クリックで退勤時刻が記録される。出勤打刻後のみ可能。 |
| US-3 | 社員として、勤怠履歴を確認したい | 月単位で出退勤時刻・勤務時間・残業時間を一覧表示できる。 |
| US-4 | 社員として、打刻を修正したい | 月次承認前の打刻を自由に修正できる（承認不要）。承認済み月は修正不可。 |
| US-5 | 社員として、休憩時間を追加記録したい | 自動控除に加え、追加の休憩を手動記録できる。 |
| US-6 | 社員として、有給休暇を申請したい | 残日数を確認し、日付を指定して申請できる。半日単位も可能。 |
| US-7 | 社員として、月次勤怠を提出したい | 月の勤怠を確認し、管理者へ承認申請できる。 |

### 管理者（管理職）

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-8 | 管理者として、部下の勤怠一覧を確認したい | メイン部署配下のメンバーの出勤状況・勤務時間を一覧で把握できる。 |
| US-9 | 管理者として、月次勤怠を承認/差戻ししたい | 部下の月次提出を確認し、承認（ロック）または差戻し（コメント付き）できる。 |
| US-10 | 管理者として、有給申請を承認/却下したい | 部下の有給申請を確認し、承認または却下できる。 |
| US-11 | 管理者として、月次レポートを CSV 出力したい | 出勤日数・総労働時間・残業時間・有給消化日数を含む CSV をダウンロードできる。 |

### システム管理者

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-12 | システム管理者として、社員を管理したい | 社員の追加・編集・無効化、ロール割当、部署割当ができる。 |
| US-13 | システム管理者として、部署を管理したい | 階層構造の部署を追加・編集・無効化できる。 |
| US-14 | システム管理者として、カレンダーを管理したい | 土日祝日・会社休日を設定できる。 |

## 勤務ルール

| 項目 | 仕様 |
|------|------|
| 所定労働時間 | 7時間30分 |
| 勤務形態 | フレックスタイム（コアタイムなし）。所定時間未満でも打刻可能、注意表示のみ |
| 休憩（自動控除） | 12:00〜13:00。適用条件: その時間帯に勤務中 かつ 当日の勤務が6時間超 |
| 休憩（手動） | 追加の休憩時間を手動記録可能 |
| 残業 | 所定時間超過を自動計算。36協定準拠（月45時間/年360時間） |
| 36協定超過時 | 警告表示のみ（打刻は拒否しない） |
| 打刻 | 1日1回（出勤・退勤のペア）。中抜けは休憩扱い |
| 打刻修正 | 月次承認前は自由に修正可能（承認ワークフローなし）。承認後はロック |
| 遅刻・早退 | コアタイムがないため概念なし |

---

## 月次承認ワークフロー

```
社員: 勤怠修正 → 提出 → [承認済み: ロック]
                  ↑         ↓ 差戻し（コメント付き）
                  └──── 修正 ←┘
```

| 項目 | 仕様 |
|------|------|
| 提出 | 社員が月の勤怠を確認し「提出」操作を行う |
| 承認者 | メイン部署の上長（管理者） |
| 承認効果 | その月の打刻がロック（修正不可） |
| 差戻し | コメント付与可能。回数制限なし |
| 管理者自身 | 自己承認 |
| 提出期限 | なし（管理者が催促） |

## 有給休暇

| 項目 | 仕様 |
|------|------|
| 付与日数 | 年間20日 |
| 付与タイミング | 4月一斉付与 |
| 繰越上限 | 最大20日 |
| 取得単位 | 全日 / 半日 |
| 承認者 | メイン部署の上長。管理者は自己承認 |

---

## 組織構造

| 項目 | 仕様 |
|------|------|
| 部署構造 | 階層型（本部→部→課 等） |
| 所属 | 社員はメイン部署（1つ）+ 兼務部署（0以上） |
| 勤怠管理の紐付け | メイン部署の管理者のみが承認権限を持つ |
| 「部下」の定義 | 部署単位で管理者を割り当て。管理者はその部署所属メンバーの勤怠を閲覧・承認できる |

---

## 非機能要件

| 項目 | 仕様 |
|------|------|
| 認証 | ID / パスワード認証 |
| パスワードリセット | なし（システム管理者が対応） |
| モバイル対応 | 不要（デスクトップ Web のみ） |
| 多言語 | 日本語のみ |
| 位置情報/IP記録 | なし |

---

## 実装優先度

| フェーズ | スコープ |
|---------|---------|
| **Phase 1（最優先）** | 認証 (ID/PW) + 出退勤打刻 (US-1, US-2) + 勤怠履歴閲覧 (US-3) |
| Phase 2 | 打刻修正 (US-4) + 休憩記録 (US-5) + 勤務時間計算（残業・36協定警告） |
| Phase 3 | 社員マスタ (US-12) + 部署管理 (US-13) + カレンダー管理 (US-14) |
| Phase 4 | 月次承認 (US-7, US-9) + 有給管理 (US-6, US-10) + 管理者一覧 (US-8) + レポート (US-11) |

---

## 月次レポート項目

- 出勤日数
- 総労働時間
- 残業時間（月累計）
- 有給消化日数
- 有給残日数
- 休日出勤日数

※ コアタイムなしのため遅刻・早退は計上しない

---

## 用語集

| 用語 | 定義 |
|------|------|
| 打刻 | 出勤または退勤の時刻を記録する行為 |
| 所定労働時間 | 1日あたりの標準勤務時間（7時間30分） |
| 残業 | 所定労働時間を超過した勤務時間 |
| 36協定上限 | 月45時間・年360時間（特別条項なしの場合） |
| 月次承認 | 社員が月の勤怠を提出し、管理者が承認する手続き。承認後は打刻ロック |
| メイン部署 | 社員の主たる所属部署。勤怠承認はこの部署の上長が行う |
| 兼務部署 | メイン以外の所属部署。勤怠承認権限には影響しない |
